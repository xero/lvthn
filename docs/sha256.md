# sha256.ts

SHA-256 — 256-bit cryptographic hash function (FIPS 180-4).

---

## Overview

`sha256.ts` exports a single class, `SHA256`, implementing the SHA-256 hash function
(NIST FIPS 180-4). SHA-256 produces a 32-byte (256-bit) digest.

The implementation uses a 512-bit (64-byte) block buffer. Message scheduling and
compression are performed in 64 rounds per block using the standard SHA-256 constant
table `K[0..63]` and initial hash values `H[0..7]`. The message length counter is a
two-word `Uint32Array` tracking the total number of bits processed.

**API pattern:** `SHA256` supports both a one-shot `hash()` method and an incremental
`init()`/`update()`/`digest()` API for processing large or streamed messages.

---

## Security Notes

**SHA-256 is a collision-resistant hash, not a MAC.** Do not use a bare SHA-256 hash
to authenticate data. Use `HMAC_SHA256` (see [hmac.ts](./hmac.md)) when you need a
message authentication code.

**Output is big-endian.** The `digest()` method serialises the eight 32-bit state words
in big-endian byte order, as specified by FIPS 180-4.

**State is reset after `digest()`.** `digest()` calls `init()` internally before
returning, leaving the object ready for reuse. This means you cannot call `digest()`
twice to get two copies of the same hash — call `hash()` on a fresh instance if you
need to repeat a hash.

**The `update()` counter uses 64-bit arithmetic via two `Uint32Array` words.** Maximum
hashable message length is 2^64 − 1 bits (≈ 2.3 × 10^18 bytes), matching the SHA-256
standard. JavaScript's 32-bit integer overflow handling is relied upon for the carry
propagation in the bit counter.

## API Reference

### `class SHA256`

Implements the `Hash` interface.

#### `constructor()`

No parameters. Initialises K constants and calls `init()`.

**Example:**
```typescript
import { SHA256 } from 'leviathan';
const sha = new SHA256();
```

---

#### `init(): SHA256`

Resets the hash state to initial values (H0–H7 per FIPS 180-4), clears the buffer,
and zeroes the bit counter. Must be called before reusing an instance after a `digest()`
has already been obtained (though `digest()` calls `init()` automatically).

**Returns:** `this` (chainable)

---

#### `update(msg?: Uint8Array): SHA256`

Feeds message bytes into the hash. Processes complete 64-byte blocks as they fill;
partial blocks remain buffered. May be called multiple times.

**Parameters:**
- `msg` — message bytes; defaults to empty if omitted

**Returns:** `this` (chainable)

---

#### `digest(msg?: Uint8Array): Uint8Array`

Finalises the hash and returns the 32-byte digest. Appends the standard SHA-256 padding
(0x80 byte, zero padding to 56 bytes, 64-bit big-endian message length), performs the
final compression, and serialises the eight state words.

Calls `init()` internally before returning, resetting the instance for reuse.

**Parameters:**
- `msg` — optional final message chunk; defaults to empty if omitted

**Returns:** `Uint8Array(32)` — 256-bit SHA-256 digest, big-endian

---

#### `hash(msg?: Uint8Array): Uint8Array`

One-shot convenience method. Calls `init()` then `digest(msg)`.

**Parameters:**
- `msg` — complete message; defaults to empty if omitted

**Returns:** `Uint8Array(32)`

**Example:**
```typescript
import { SHA256, Convert } from 'leviathan';

const sha  = new SHA256();
const hash = sha.hash(Convert.str2bin('abc'));
console.log(Convert.bin2hex(hash));
// ba7816bf8f01cfea414140de5dae2ec73b00361bbef0469348423f656fdb91c
```

---

#### `selftest(): boolean`

Runs a built-in 1000-iteration accumulating self-test and compares the result against
a known expected value. Returns `true` if the implementation is correct.

**Returns:** `true` if the self-test passes

---

## Usage Examples

### One-shot hash

```typescript
import { SHA256, Convert } from 'leviathan';

const sha  = new SHA256();
const hash = sha.hash(Convert.str2bin('Hello, world!'));
console.log(Convert.bin2hex(hash));
```

---

### Incremental (streaming) hash

```typescript
import { SHA256, Convert } from 'leviathan';

const sha = new SHA256();
sha.update(Convert.str2bin('chunk one — '));
sha.update(Convert.str2bin('chunk two — '));
sha.update(Convert.str2bin('chunk three'));
const hash = sha.digest();  // no extra data; finalise
```

---

### As the underlying hash for HMAC-SHA256

```typescript
import { HMAC_SHA256, Convert, Random } from 'leviathan';

const mac = new HMAC_SHA256();
const key = new Random().get(32)!;
const tag = mac.hash(key, Convert.str2bin('Authenticated message'));
```

---

## Error Conditions

- **Calling `update()` after `digest()`:** `digest()` resets state via `init()`, so
  any data passed to `update()` after a `digest()` call starts a fresh hash. This is
  not an error, but may be unexpected.
- **No argument passed to `hash()` or `digest()`:** Treated as an empty message
  (`new Uint8Array(0)`). This is valid and produces the SHA-256 hash of the empty string.
- **Null/undefined inputs:** Will throw a runtime `TypeError`.
