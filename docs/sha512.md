# sha512.ts

SHA-512 — 512-bit cryptographic hash function (FIPS 180-4).

---

## Overview

`sha512.ts` exports a single class, `SHA512`, implementing the SHA-512 hash function
(NIST FIPS 180-4). SHA-512 produces a 64-byte (512-bit) digest.

The implementation represents the 64-bit state words and round constants as pairs of
32-bit `Uint32Array` entries (high/low) to work within JavaScript's 32-bit integer
arithmetic. The message schedule uses a 1024-bit (128-byte) block. Compression runs
80 rounds (versus SHA-256's 64).

**API pattern:** Identical to `SHA256` — supports both a one-shot `hash()` method and
the incremental `init()`/`update()`/`digest()` sequence.

---

## Security Notes

**SHA-512 is a hash, not a MAC.** Do not use a bare SHA-512 hash to authenticate
messages. Use `HMAC_SHA512` (see [hmac.ts](./hmac.md)) for message authentication.

**64-bit words emulated via 32-bit pairs.** JavaScript's bitwise operators operate on
32-bit signed integers. SHA-512 requires 64-bit rotations and additions. The
implementation carries these out as paired 32-bit operations (high/low). This is correct
but slower than a native 64-bit implementation.

**Output is big-endian.** The 16 state words (8 × 64-bit = 16 × 32-bit) are serialised
high-word first, then low-word, each in big-endian byte order.

**State is reset after `digest()`.** As with `SHA256`, `digest()` calls `init()`
before returning. The object is immediately ready for reuse.

**`selftest()` known-answer string:** The self-test accumulates 1000 SHA-512 hashes and
checks against a hardcoded 128-character hex string. Note the expected value in the
source has 127 hex characters (63.5 bytes), which is a one-character truncation. This
does not affect production hash correctness — only the self-test result would fail if the
accumulation matched exactly 64 bytes with a leading zero nibble.

## API Reference

### `class SHA512`

Implements the `Hash` interface.

#### `constructor()`

No parameters. Initialises K constants (80 × 64-bit values as 160 × 32-bit words) and
calls `init()`.

**Example:**
```typescript
import { SHA512 } from 'leviathan';
const sha = new SHA512();
```

---

#### `init(): SHA512`

Resets the hash state to the SHA-512 initial hash values (H0–H7 per FIPS 180-4, stored
as 16 × 32-bit words), clears the 128-byte buffer, and zeroes the bit counter.

**Returns:** `this` (chainable)

---

#### `update(msg?: Uint8Array): SHA512`

Feeds message bytes into the hash. Processes complete 128-byte blocks as they fill;
partial blocks remain buffered. May be called multiple times.

**Parameters:**
- `msg` — message bytes; defaults to empty if omitted

**Returns:** `this` (chainable)

---

#### `digest(msg?: Uint8Array): Uint8Array`

Finalises the hash and returns the 64-byte digest. Appends the standard SHA-512 padding
(0x80 byte, zero padding to 112 bytes, 128-bit big-endian message length), performs the
final compression, and serialises the 16 state words.

Calls `init()` internally before returning.

**Parameters:**
- `msg` — optional final message chunk; defaults to empty if omitted

**Returns:** `Uint8Array(64)` — 512-bit SHA-512 digest, big-endian

---

#### `hash(msg?: Uint8Array): Uint8Array`

One-shot convenience method. Calls `init()` then `digest(msg)`.

**Parameters:**
- `msg` — complete message; defaults to empty if omitted

**Returns:** `Uint8Array(64)`

**Example:**
```typescript
import { SHA512, Convert } from 'leviathan';

const sha  = new SHA512();
const hash = sha.hash(Convert.str2bin('abc'));
console.log(Convert.bin2hex(hash));
// ddaf35a193617aba...
```

---

#### `selftest(): boolean`

Runs a built-in 1000-iteration accumulating self-test.

**Returns:** `true` if the implementation is correct

---

## Usage Examples

### One-shot hash

```typescript
import { SHA512, Convert } from 'leviathan';

const sha  = new SHA512();
const hash = sha.hash(Convert.str2bin('Hello, world!'));
console.log(Convert.bin2hex(hash));
```

---

### Incremental hash

```typescript
import { SHA512, Convert } from 'leviathan';

const sha = new SHA512();
sha.update(Convert.str2bin('part one — '));
sha.update(Convert.str2bin('part two'));
const hash = sha.digest();  // finalise
```

---

### As the underlying hash for HMAC-SHA512

```typescript
import { HMAC_SHA512, Convert, Random } from 'leviathan';

const mac = new HMAC_SHA512();
const key = new Random().get(64)!;  // 64-byte key matches SHA-512 hashSize
const tag = mac.hash(key, Convert.str2bin('Authenticated message'));
```

---

## Error Conditions

- **`update()` after `digest()`:** `digest()` resets state, so subsequent `update()`
  calls begin a fresh hash. This is valid but potentially confusing.
- **No argument to `hash()` or `digest()`:** Produces the SHA-512 hash of the empty
  string. This is valid.
- **Null/undefined inputs:** Will throw a runtime `TypeError`.
