# x25519.ts

Curve25519 (ECDH key exchange) and Ed25519 (digital signatures).

---

## Overview

`x25519.ts` exports two classes:

| Class | Protocol | Use |
|-------|----------|-----|
| `Curve25519` | X25519 / ECDH | Key generation and Diffie-Hellman shared secret |
| `Ed25519` | EdDSA | Digital signatures (sign + verify) |

Both are built on the Curve25519 elliptic curve over GF(2^255 − 19). The implementation
is derived from `curve255js` and `jodid25519`, using field elements represented as 16-limb
`Int32Array` values in radix 2^16.

**Key design:** leviathan stores Ed25519 secret keys as the original 32-byte seed, not as
the extended 64-byte (seed ∥ public key) format used by some libraries (libsodium, NaCl).
The public key is derived from the seed on demand via SHA-512 hashing and key hardening.

---

## Security Notes

**Signature verification uses `constantTimeEqual`.** The final step of `Ed25519.verify()`
compares the recomputed R point against the signature's R component using `constantTimeEqual`.
This prevents timing-oracle attacks on signature verification that could leak information
about the secret key.

**Curve point comparison uses `constantTimeEqual`.** `neq25519()` (used in `unpackNeg`)
packs both field elements to canonical byte form and compares with `constantTimeEqual`
rather than direct integer comparison, preventing timing side-channels during the point
decompression step.

**Key hardening (`clamp25519`):** Both `Curve25519.generateKeys()` and `Ed25519.generateKeys()`
apply the standard Curve25519 clamp to the scalar:
- Clear bits 0, 1, 2 of byte 0 (`sk[0] &= 0xf8`): ensures the scalar is a multiple of 8
  (the cofactor of Curve25519), mitigating small-subgroup attacks.
- Clear bit 255 of byte 31 (`sk[31] &= 0x7f`): prevents the scalar from exceeding the
  field prime.
- Set bit 254 of byte 31 (`sk[31] |= 0x40`): ensures the top bit is always set,
  providing a constant-time ladder property.

**Not constant-time in JavaScript.** JavaScript engines provide no formal constant-time
guarantees. The Curve25519 ladder (`sel25519` / `cswap`) avoids data-dependent branches
on the secret scalar bits, which eliminates the most practical side-channel vector.
However, a formally proven constant-time guarantee requires WebAssembly or native code.

**Ed25519 key storage:** The secret key stored and returned is the 32-byte seed, not the
hashed and clamped scalar. The hash expansion is performed internally during `sign()` and
`generateKeys()`. Do not pass the 64-byte extended key format — only 32-byte seeds are
accepted.

## API Reference

### `class Curve25519`

Provides X25519 Diffie-Hellman key exchange.

#### `constructor()`

No parameters. Initialises curve constants (D, D2, I, _121665) and base point (`_9`).

---

#### `generateKeys(seed: Uint8Array): { sk: Uint8Array, pk: Uint8Array } | undefined`

Generates a Curve25519 keypair from a 32-byte random seed.

The seed is clamped to produce the secret scalar, then multiplied by the base point
(u=9) to produce the public key.

**Parameters:**
- `seed` — 32 bytes of cryptographically random data (from `Random.get(32)`)

**Returns:** `{ sk: Uint8Array(32), pk: Uint8Array(32) }` — clamped secret key and
public key, or `undefined` if `seed.length !== 32`

**Example:**
```typescript
import { Curve25519, Random } from 'leviathan';

const rng   = new Random();
const curve = new Curve25519();

const seed  = rng.get(32)!;
const keys  = curve.generateKeys(seed)!;
// keys.sk — secret key (keep secret)
// keys.pk — public key (send to counterpart)
```

---

#### `scalarMult(sk: Uint8Array, pk: Uint8Array): Uint8Array`

Computes the Diffie-Hellman shared secret: `sk × pk` on Curve25519.

**Parameters:**
- `sk` — 32-byte secret key (from `generateKeys().sk`)
- `pk` — 32-byte public key of the remote party

**Returns:** `Uint8Array(32)` — shared secret (use as input to a KDF, not directly as
a symmetric key)

**Example:**
```typescript
// Alice computes shared secret using her sk and Bob's pk:
const sharedA = curve.scalarMult(aliceKeys.sk, bobKeys.pk);

// Bob computes the same shared secret using his sk and Alice's pk:
const sharedB = curve.scalarMult(bobKeys.sk, aliceKeys.pk);

// sharedA === sharedB (Diffie-Hellman property)
```

---

#### `selftest(): boolean`

Verifies key generation and scalar multiplication against known test vectors from
RFC drafts. Returns `true` if correct.

---

### `class Ed25519`

Provides Ed25519 digital signatures.

#### `constructor()`

No parameters. Initialises the base point coordinates X, Y, and the group order L.

---

#### `generateKeys(seed: Uint8Array): { sk: Uint8Array, pk: Uint8Array } | undefined`

Generates an Ed25519 keypair from a 32-byte seed.

Internally: hashes the seed with SHA-512, clamps the first 32 bytes of the hash as the
scalar, and multiplies by the Ed25519 base point to produce the public key. The returned
`sk` is the original seed, not the hashed scalar.

**Parameters:**
- `seed` — 32 bytes of cryptographically random data

**Returns:** `{ sk: Uint8Array(32), pk: Uint8Array(32) }`, or `undefined` if
`seed.length !== 32`

**Example:**
```typescript
import { Ed25519, Random } from 'leviathan';

const rng   = new Random();
const ed    = new Ed25519();

const seed = rng.get(32)!;
const keys = ed.generateKeys(seed)!;
// keys.sk — secret key seed (keep secret)
// keys.pk — public key (share openly)
```

---

#### `sign(msg: Uint8Array, sk: Uint8Array, pk: Uint8Array): Uint8Array | undefined`

Signs a message using an Ed25519 secret key.

Follows the Ed25519 signing procedure (RFC 8032 §5.1.6): deterministic nonce r derived
from `SHA512(h[32..63] ∥ msg)`, signature `(R, s)` where `R = r × B` and
`s = (r + k × a) mod l`.

**Parameters:**
- `msg` — message bytes to sign (any length)
- `sk` — 32-byte secret key seed (from `generateKeys().sk`)
- `pk` — 32-byte public key (from `generateKeys().pk`)

**Returns:** `Uint8Array(64)` — 64-byte signature, or `undefined` if `sk.length !== 32`
or `pk.length !== 32`

---

#### `verify(msg: Uint8Array, pk: Uint8Array, sig: Uint8Array): boolean`

Verifies an Ed25519 signature.

Recomputes the expected R point and compares to `sig[0..31]` using `constantTimeEqual`.

**Parameters:**
- `msg` — the message that was signed
- `pk` — 32-byte public key of the signer
- `sig` — 64-byte signature

**Returns:** `true` if the signature is valid, `false` otherwise

**Example:**
```typescript
import { Ed25519, Convert } from 'leviathan';

const ed  = new Ed25519();
const sig = ed.sign(msg, keys.sk, keys.pk)!;

if (!ed.verify(msg, keys.pk, sig)) {
  throw new Error('Signature verification failed');
}
```

---

#### `selftest(): boolean`

Verifies sign and verify against three known test vectors from
draft-josefsson-eddsa-ed25519-03. Returns `true` if all pass.

---

## Usage Examples

### X25519 Diffie-Hellman key exchange

```typescript
import { Curve25519, Random } from 'leviathan';

const rng   = new Random();
const curve = new Curve25519();

// Alice generates her keypair
const alice = curve.generateKeys(rng.get(32)!)!;

// Bob generates his keypair
const bob   = curve.generateKeys(rng.get(32)!)!;

// Alice and Bob exchange public keys, then compute the shared secret
const sharedA = curve.scalarMult(alice.sk, bob.pk);
const sharedB = curve.scalarMult(bob.sk, alice.pk);

// sharedA equals sharedB — use as input to HKDF/PBKDF2 to derive a symmetric key
// Do not use the shared secret directly as a cipher key without a KDF step
```

---

### Ed25519 sign and verify

```typescript
import { Ed25519, Random, Convert } from 'leviathan';

const rng = new Random();
const ed  = new Ed25519();

// Key generation
const keys = ed.generateKeys(rng.get(32)!)!;

// Signing
const msg = Convert.str2bin('Document to authenticate');
const sig = ed.sign(msg, keys.sk, keys.pk)!;

// Verification
if (!ed.verify(msg, keys.pk, sig)) {
  throw new Error('Invalid signature');
}
console.log('Signature valid');

// A tampered message or signature will fail
const tampered = new Uint8Array(sig);
tampered[0] ^= 0x01;
console.log(ed.verify(msg, keys.pk, tampered));  // false
```

---

## Error Conditions

- **`generateKeys()` with `seed.length !== 32`:** Returns `undefined`. Always validate.
- **`sign()` with `sk.length !== 32` or `pk.length !== 32`:** Returns `undefined`.
- **`verify()` with `sig.length !== 64` or `pk.length !== 32`:** Returns `false`.
- **`verify()` with a valid point-decompression failure (`unpackNeg` returns -1):**
  Returns `false`. The public key bytes do not represent a valid curve point.
- **Null/undefined inputs:** Will throw a runtime `TypeError`.
